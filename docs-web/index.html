Sobre o Projeto

Sistema de gateway IoT para integração de múltiplos protocolos de sensores (LoRa, Zigbee, Sigfox, RFID) com comunicação via MQTT para processamento e armazenamento de dados.

Stack Tecnológico

Python 3.x MQTT / Paho-MQTT Eclipse Mosquitto LoRa Zigbee Sigfox RFID JSON
Início / Proposta Arquitetônica

Proposta Arquitetônica

Visão completa da arquitetura do sistema IoT Gateway

Visão Geral

O sistema IoT Gateway é uma solução modular para integração de múltiplos protocolos de sensores IoT, centralizando a comunicação através do protocolo MQTT e fornecendo uma interface unificada para processamento de dados.

Camadas da Arquitetura

1. Camada de Sensores

  • LoRa Sensor: Comunicação de longo alcance, baixo consumo
  • Zigbee Sensor: Redes mesh, automação residencial
  • Sigfox Sensor: Comunicação global, IoT massivo
  • RFID Sensor: Identificação por radiofrequência

2. Camada de Gateway

  • MQTT Client: Cliente MQTT para publicação e subscrição
  • Message Formatter: Formatação e validação de mensagens
  • Config Loader: Gerenciamento de configurações
  • Gateway Core: Orquestração dos sensores

3. Camada de Broker MQTT

  • Eclipse Mosquitto: Broker MQTT (localhost:1883)
  • Topics: sensores/+/dados (publicação por tipo de sensor)
  • QoS: Quality of Service Level 1 (at least once)

4. Camada de Aplicação

  • Subscribers: Processamento de mensagens por tópico
  • Data Storage: Persistência de dados (futuro)
  • API REST: Interface para consulta (futuro)

Fluxo de Dados

Sensor → Gateway → MQTT Broker → Subscriber → Processamento
  │         │           │              │            │
  │         │           │              │            ├─→ Storage
  │         │           │              │            └─→ Analytics
  │         │           │              │
  │         │           └──────────────┴─── Topics específicos
  │         │
  │         └─── Formatação e validação
  │
  └─── Coleta de dados físicos

Características Principais

  • Suporte multi-protocolo (LoRa, Zigbee, Sigfox, RFID)
  • Comunicação assíncrona via MQTT
  • Arquitetura modular e extensível
  • Formatação padronizada de mensagens JSON
  • Configuração centralizada via INI
  • Simuladores para testes sem hardware

Início / Integrações

Integrações

Protocolos suportados e suas características

LoRa (Long Range)

Características:

  • Alcance: até 15 km (rural)
  • Frequência: 868 MHz (EU) / 915 MHz (US)
  • Taxa: 0.3 - 50 kbps
  • Consumo: Muito baixo

Uso: Sensores remotos, agricultura, smart cities

Topic MQTT: sensores/lora/dados

Zigbee

Características:

  • Alcance: 10-100 metros
  • Frequência: 2.4 GHz
  • Taxa: 250 kbps
  • Topologia: Mesh

Uso: Automação residencial, iluminação inteligente

Topic MQTT: sensores/zigbee/dados

Sigfox

Características:

  • Alcance: até 50 km
  • Frequência: 868 MHz
  • Taxa: 100 bps
  • Mensagens: 140 msg/dia

Uso: Rastreamento, medição, alarmes

Topic MQTT: sensores/sigfox/dados

RFID

Características:

  • Alcance: 0.1 - 10 metros
  • Frequência: 125 kHz - 2.4 GHz
  • Taxa: Instantânea
  • Sem bateria (passivo)

Uso: Controle de acesso, inventário, logística

Topic MQTT: sensores/rfid/dados

Formato de Mensagem Padrão

{
  "tipo_sensor": "lora|zigbee|sigfox|rfid",
  "sensor_id": "unique_sensor_identifier",
  "timestamp": "2025-10-15T14:30:00Z",
  "dados": {
    "temperatura": 23.5,

    "bateria": 87
  },
  "metadata": {
    "localizacao": "Sala A",
    "versao": "1.0.0"
  }
}

Início / Operações

Operações

Guias operacionais e procedimentos

Inicialização do Sistema

  1. Iniciar o broker Mosquitto: mosquitto -v
  2. Ativar ambiente virtual: .venv\Scripts\activate
  3. Executar o gateway: python backend/gateway/gateway.py
  4. Executar simuladores: python tests/test_simuladores.py

Monitoramento

  • Logs do Mosquitto: Verificar console do broker
  • Logs do Gateway: Saída padrão do Python
  • Status dos sensores: Verificar última comunicação
